# Contributing to vikrant-OBLITERATUS

Thank you for your interest in contributing! We welcome contributions from the research community.

## Priority Areas

1. **New Abliteration Methods**: Novel direction extraction techniques
2. **Analysis Modules**: Improved refusal geometry analysis
3. **Voice Model Support**: Expanding audio-capable LLM coverage
4. **Security**: Threat database contributions
5. **Documentation**: Tutorials, examples, research papers

## Development Setup

```bash
git clone https://github.com/vikrant-project/vikrant-OBLITERATUS.git
cd vikrant-OBLITERATUS
pip install -e .[dev]
```

## Code Standards

- **Formatting**: Black (line length 100)
- **Linting**: Ruff
- **Type Hints**: Required for all functions
- **Tests**: Pytest coverage >80%
- **Docstrings**: Google style

## Contribution Workflow

1. Fork the repository
2. Create feature branch: `git checkout -b feature/amazing-method`
3. Make changes with tests
4. Run tests: `pytest tests/`
5. Format code: `black vikrant_obl/`
6. Commit: `git commit -m 'Add amazing method'`
7. Push: `git push origin feature/amazing-method`
8. Open Pull Request

## Pull Request Guidelines

- **Title**: Clear, descriptive (e.g., "Add cross-lingual abliteration method")
- **Description**: Explain what, why, how
- **Tests**: Include unit tests for new features
- **Documentation**: Update README if adding features

## Community

- **Issues**: Report bugs, request features
- **Discussions**: Research questions, design decisions
- **Discord**: Join our research community (link in README)

## Code of Conduct

Be respectful, collaborative, and inclusive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).
